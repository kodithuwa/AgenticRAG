# .NET 10 RAG Console App

A minimal but complete **Retrieval-Augmented Generation** application built with:

| Layer | Technology |
|---|---|
| Language / Runtime | C# 13 / .NET 10 |
| ORM | Entity Framework Core 10 |
| Vector DB | PostgreSQL 16 + pgvector |
| EF pgvector driver | `Pgvector.EntityFrameworkCore` |
| AI orchestration | `Microsoft.SemanticKernel` 1.30 |
| Embeddings | OpenAI `text-embedding-3-small` (1536 dims) |
| Chat | OpenAI `gpt-4o-mini` |

## Quick Start

### 1. Start PostgreSQL with pgvector

```bash
docker compose up -d
```

### 2. Configure secrets

Edit `appsettings.json` (or set environment variables):

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Host=localhost;Port=5432;Database=rag_db;Username=postgres;Password=your_password"
  },
  "OpenAI": {
    "ApiKey": "sk-...",
    "EmbeddingModel": "text-embedding-3-small",
    "ChatModel": "gpt-4o-mini"
  }
}
```

> **Tip:** Use `dotnet user-secrets` or environment variables (`OpenAI__ApiKey=sk-...`) to keep secrets out of source control.

### 3. Apply EF migrations

```bash
dotnet tool install --global dotnet-ef
dotnet ef migrations add InitialCreate
dotnet ef database update
```

### 4. Run

```bash
dotnet run
```

## Key Design Decisions

- **IVFFlat index** on the embedding column (`lists=100`) for fast ANN at scale. Tune `lists` в‰€ `sqrt(row_count)`.  
- **Cosine similarity** (`vector_cosine_ops`) вЂ” best for normalised text embeddings.  
- **Chunk overlap** (default 50 words) avoids cutting context at chunk boundaries.  
- **`DatabaseInitializer`** calls `MigrateAsync` at startup вЂ” zero manual migration steps after first deploy.
